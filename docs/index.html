8 disk.s10hw2 — a Solaris 10 HW2 disk image used as the reference/known-good test image for both this project (qemu-sparc64) and the original Legion OpenSPARC T1 emulator; not an installer ISO like the other columns, no separate download link.

9 Not actually run against these cells -- inferred from snv_134/256's established pattern of bop_alloc_chunk OOM panics on the two other images already tested against this build (snv_134 ai ISO, Solaris 10u8); 256MB is already known to be too small to boot any of these installer images.

10 Every cell previously recorded here has been reverted to untested: the build had a memory-config bug (hv-config GUEST() memsize left at 1024M instead of being bumped to 2048M alongside the MD's mblock.size), so every prior tlb_fix/2048 result was actually tested against a hypervisor-granted 1024M, not the claimed 2048M. Fixed 2026-07-17; all cells need re-running against the corrected build.

MD/hv-config builds under test

  • OpenSPARC_T1_rebuild — Sun's unmodified 2006 sources, rebuilt with our mdgen; byte-identical to the untouched OpenSPARC_T1_original reference in bin/, so same expected boot behavior
  • 1GiB-snv_77 (1GiB-experimental) — externally-sourced prebuilt blobs, not built from src/
  • snv_134/256, snv_134/1024, snv_134/2048 — memory/cpus/cache + mmu-* fix applied (see patches/snv_134/), 256/1024/2048 MiB variants
  • tlb_fix/1024, tlb_fix/2048 — snv_134/1024 and snv_134/2048 plus one added tlb node (see src/tlb_fix/), fixing the sfmmu_cpu_init panic ("failed to get MMU CTX domain index") hit by Solaris 10u10+/Solaris 11 on the snv_134 builds. tlb_fix/4096 also exists (a further memory bump past 2048, to check whether a bop_alloc_chunk OOM panic is genuine memory exhaustion) but is reference-only, not part of the boot-status table below
See README for the repo layout (src/bin/patches/fw_blobs) and build instructions.
