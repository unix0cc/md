MD/hv-config builds under test

  • OpenSPARC_T1_rebuild — Sun Microsystems original unmodified 2006 source code from OpenSparc T1 Legion code (OpenSPARCT1_Arch.1.5.tar.bz2, reference), rebuilt with mdgen; byte-identical to the untouched OpenSPARC_T1_original reference in bin/, so same expected boot behavior
  • 1GiB-snv_77 (1GiB-experimental) — externally-sourced prebuilt blobs, not built from src/
  • t1_boot_fix/256, t1_boot_fix/1024, t1_boot_fix/2048 — memory/cpus/cache + mmu-* fix applied (see patches/t1_boot_fix/), 256/1024/2048 MiB variants
  • tlb_fix/1024, tlb_fix/2048 — t1_boot_fix/1024 and t1_boot_fix/2048 plus one added tlb node (see src/tlb_fix/), fixing the sfmmu_cpu_init panic ("failed to get MMU CTX domain index") hit by Solaris 10u10+/Solaris 11 on the t1_boot_fix builds. tlb_fix/4096 also exists (a further memory bump past 2048, to check whether a bop_alloc_chunk OOM panic is genuine memory exhaustion) but is reference-only, not part of the boot-status table below
See README for the repo layout (src/bin/patches/fw_blobs) and build instructions.
