Usage

-m <memsize> is QEMU's guest memory size flag. Each configuration is memory-deterministic: the MD/HV binaries under a given /git/md <memsize> path are built for that one specific memory size, so for a successful boot -m's value must match the <memsize> in the -L path exactly. A mismatch (e.g. -L /git/md/bin/tlb_fix/1024 with -m 2048) means the guest is told it has memory the MD never declared, and will fail to boot correctly.

Syntax:

./qemu-system-sparc64 -M niagara -L <md_dir>/<memsize> -nographic -m <memsize> -drive if=pflash,readonly=on,file=<path_to_file>/<iso_file.iso>

<md_dir>/<memsize> is a directory containing the MD/HV binaries (1up-md.bin, 1up-hv.bin) and the required firmware binaries (netcons, nvram1, openboot.bin, q.bin), e.g. /git/md/bin/tlb_fix/1024.

Example:

./qemu-system-sparc64 -M niagara -L /git/md/bin/tlb_fix/1024 -nographic -m 1024 -drive if=pflash,readonly=on,file=/zroot/iso/sol-10-u8-ga-sparc-dvd.iso

Build Procedure

The build procedure is described here: https://github.com/unix0cc/md/. Requires a toolkit (mdgen/mdlint) used for compiling the .bin files (1up-md.bin, 1up-hv.bin).
